# Требования: bg-club-bot

Telegram-бот + Mini App для клуба настольных игр.

## Стек

- **Bot:** Python, aiogram 3
- **API:** FastAPI
- **БД:** PostgreSQL
- **Кэш / FSM:** Redis
- **Mini App:** TBD (React + Vite — на этапе MVP 5)
- **Деплой:** Docker, VPS

## Роли

| Роль | Возможности |
|------|-------------|
| Гость / пользователь | Инфо о клубе, игры, лента, виртуальные столы, бронь физ. стола, профиль, абонемент (просмотр) |
| Админ | CRUD игр, ивентов, объявлений; абонементы вручную; check-in; удаление виртуальных столов; управление бронями |

Все пользователи могут создавать виртуальные столы и присоединяться к ним.

## Модули

### Публичное

- **О клубе** — адрес, часы, правила
- **Каталог игр** — админ ведёт список; при создании виртуального стола — выбор из каталога или «своя игра»
- **Объявления** — новости без обязательной даты события
- **Ивенты (календарь)** — события с датой/временем; **без записи** через бота, только информирование
- **Лента** — UI: ближайшие ивенты + свежие объявления (отдельные сущности в БД)

### Виртуальные игровые столы

Социальный matchmaking: «ищу компанию в игру». **Не связаны** с бронью физического стола в зале.

**Создание стола:**

- игра из каталога **или** «Своя игра» + текстовое поле
- дата и время
- макс. число игроков (создатель = 1 место)
- комментарий (опционально)

**Правила:**

- создавать и присоединяться могут все
- лимита на кол-во созданных столов нет
- админ может удалить любой стол

**Уведомления (через бота):**

| Событие | Получатели |
|---------|------------|
| Кто-то присоединился | Создатель |
| Стол полный (все места заняты) | Все участники + админ («встреча состоится») |
| Напоминание за N ч до начала | Все участники |
| Отмена / удаление админом | Все участники |

### Бронь физического стола

Отдельный модуль. Заявка: время, стол (если есть), кол-во человек.

- **без автоматических лимитов** и блокировки пересечений
- при большом числе броней админ связывается с людьми вручную
- статусы: `submitted` → админ вручную `confirmed` / `cancelled`

### Абонементы

- админ выдаёт, продлевает, «гасит» **вручную**
- автооплата — не в MVP
- виртуальные столы и бронь доступны всем, независимо от абонемента

### Учёт посещений (check-in)

Два пути:

1. **Через абонемент** — визит + списание/учёт по абонементу
2. **Разовый визит** — гость без абонемента, админ отмечает отдельно

Оба пишутся в `visits` с источником `membership` | `walk_in`.

### Исключено из scope

- бар, склад, магазин, учёт напитков и снеков

## Bot vs Mini App

| Bot | Mini App |
|-----|----------|
| `/start`, регистрация | Основной UI |
| Menu Button → открыть приложение | Каталог, столы, брони, календарь, админка |
| Все push-уведомления | Формы, списки, календарь |

## MVP-этапы

1. Backend + БД + бот (`/start`, menu) + базовый Mini App: клуб, игры, лента
2. Виртуальные столы + все нотификации
3. Ивенты, объявления, календарь
4. Бронь физ. стола + абонементы + check-in
5. Полная админка в Mini App
